# Task 5: Prisma 7 unsupported/deprecated inventory

Generated by `worker-3` on 2026-05-26. Scope is a Phase 1 proof-spike
inventory only: no broad source edits, no runtime port, and no compatibility

## Official current-version anchors

- Prisma ORM 7 upgrade guide: MongoDB is not yet supported; Node minimum is
  20.19.0; Prisma ships as ESM; `output` is required for the Prisma Client
  generator; datasource URL / direct URL / shadow DB URL move to
  `prisma.config.ts`; driver adapters are required for direct DB connections;
  automatic env loading changed; metrics, selected CLI flags, and selected env
  vars were removed:
  <https://www.prisma.io/docs/guides/upgrade-prisma-orm/v7>
- Prisma MongoDB quickstart: v7 MongoDB support is not available yet; MongoDB
  users should use Prisma ORM v6.19 for now:
  <https://www.prisma.io/docs/prisma-orm/quickstart/mongodb>
- Prisma metrics docs: the `metrics` preview feature was removed as of Prisma
  ORM v7.0.0; replacement guidance is database-driver native metrics or
  OpenTelemetry:
  <https://www.prisma.io/docs/v6/orm/prisma-client/observability-and-logging/metrics>
- Prisma Client generation docs: Prisma ORM v7 requires a generator `output`
  path:
  <https://www.prisma.io/docs/orm/prisma-client/setup-and-configuration/generating-prisma-client/>

## Release-blocking test matrix implied by this inventory

1. **A1 generator fixture:** Prisma 7 package versions, Node >=20.19, ESM package config, `prisma.config.ts`, and explicit generator `output`.
2. **MongoDB negative/skip fixture:** v7 path rejects/skips MongoDB with explicit messaging, while v6 maintenance tests remain separate.
3. **Metrics v7 behavior:** `get_metrics()` is removed/unsupported under v7, or a replacement observability bridge is proven; no `/metrics` parity assumption.
4. **CLI migration fixture:** `db push` / migrate flow calls `prisma generate` explicitly and does not rely on removed `--skip-generate` behavior.
5. **Config/env fixture:** database URL is loaded through `prisma.config.ts` with explicit dotenv import, not only schema `env()` plus Python-side `.env` loading.
6. **A2 runtime proof:** either official JS Client + driver adapter bridge works end to end, or a documented non-JS Prisma 7 query boundary is proven; private Rust endpoints do not count.
